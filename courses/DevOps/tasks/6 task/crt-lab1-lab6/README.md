# Практическое задание 6

## Дано

Пример **docker-compose.yml** с основным приложением и вспомогательным. Пример **docker-compose-infra.yml** c сервисами *rmq, redis, postgres, zookeeper, haproxy, minio.* Примеры **докерфайлов**

## Задание

- Сделать новый **docker-compose.yml** для своего сервиса , либо “довести до ума” данный в примере до полноценного вида
- Сделать (или закончить на основе примера) логическое разделение всех сервисов и инфры - разные зависимости и файлы переменных, докерфайлы, композы
- Сделать **init-контейнер** (в композе с самим приложением), который будет “наполнять” базу таблицей (таблицами). \*По возможности в коде обращаться именно к этой таблице
- Сделать **init-контейнер** (в композе с самим приложением), который будет создавать `bucket` в **Minio.** *Подсказка:* утилита `mc`
- \*Добавить кэширование **redis** в части запроса статусов из БД (`/order/<id>` из примера)
-  \*Добавить логирование в `stdout`, помимо уже имеющегося в файл

## Проверка результата

- При запуске приложения создается таблица с нужными полями в БД Postgres и бакет в Minio
- Пройти флоу тестового заказа: создать заказ через `POST /order/create`, получить статус заказа через `GET /order/<id>`
