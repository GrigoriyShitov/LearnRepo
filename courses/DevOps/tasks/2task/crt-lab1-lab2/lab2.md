# Практическое задание 2

## Дано

Пример **docker-compose.yml** с сервисом и инит-контейнером

## Задание

Сделать новый **docker-compose.yml** для своего сервиса (минимум два в композе), либо “довести до ума” данный в примере до полноценного вида. Обязательные требования:

- жесткое именование получившихся контейнеров
- автоматическая сборка образа из лежащего рядом **Dockerfile** и присваивание ему (образу) имени
- для основного приложения указать, чтоб ребутался при “падении” сам
- использование `volume` (можно монтировать хоть пустой файл)
- использование `network`
- для основного приложения переопределить `command` (можно что-то поменять, можно оставить как в **Dockerfile**)
- добавить `healthcheck`
- добавить хотя бы одну переменную среды, причем через файл **.env**
